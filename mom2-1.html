Fråga 1

  • HTML HyperText Markup Language är ett så kallat "markup language" som används för att skapa hemsidor. Markup är ett format som har taggar och element som beskriver hur dokumentet skall se ut när det presenteras för användare

  • XHTML Extensible HyperText Markup Language är ett striktare form av html som är lite hårdare med reglerna. HTML går att komma undan med vissa misstag i formatering och ändå kan sidan visas korrekt. XHTML kräver att man skriver formateringen korrekt för att sidan skall visas

  • HTML5 är HTML med fler tillgängliga element så man har mer alternativ när man skriver kod för en hemsida

  • Referens w3schools HTML
  • Referens w3schools XHTML
  • Referens w3schools HTML5

Fråga 2

  • En tag är början eller slutet på ett element. Ett paragraf element består av öppningstagen <p> och sedan någon text och sedan avslutningstaggen </p>

  • Referens webdesign.about.com

Fråga 3

  • p elementet är en text element där man skriver "vanlig brödtext". Detta stycke som du läser nu är ett paragraf element

  • html elementet är elementet som omger hela html dokumentet

  • body elementet är huvuddelen av sidan och där man lägger den mesta texten och innehållet. Det finns bara ett body element i en hemsida

  • header elementet innehåller ofta navigering på sidan och t.ex. logotyp mm.

  • h1 - h6 elementen är olika former av rubriker ju lägre nummer desto större rubrik i allmänhet

  • Referens mozilla.org

Fråga 4

  • Att inte avsluta ett element anses som ganska dålig kodningsstil. För det första så blir det svårare att läsa html koden för det andra kan det ibland bli fel på grund av det. T.ex. så kan det bli svårare att implementera css och javascript funktioner när inte det finns kompletta element. Den enda fördelen som jag hittat är att det ibland kan gå fortare att hämta en hemsida av den enkla anledningen att det finns färre tecken i filen man laddar ner. Men med tanke på hur snabb uppkoppling de flesta har nu förtiden så tycker jag personligen att det är dumt att inte använda avslutningstaggar.

  • Sedan finns det undantag. T.ex. ny rad taggen br som enbart är en tag när man vill bryta en rad och börja på nästa.

  • Referens webdesign.about.com

Fråga 6

  • Body är den största delen av en hemsida. Man kan säga huvuddelen att materialet som finns på en hemsida ligger i body elementet. Som jag skrev tidigare kan det endast finnas ett body element per sida.

  • Referens mozilla.org

Fråga 7

  • En absolut url pekar någonstans som man kan nå från vart som helst som man har uppkoppling. T.ex. https://www.miun.se/utbildning/kurser/data-och-it/datateknik/datateknik-gr-a-webbutveckling-i-75-hp/om-kursen Däremot en relativ länk pekar på en sida/fil inom den aktuella hemsidan. Som när jag länkar till denna sidan från min första sida så använder jag bara länken mom2-1.html. Det kan du ju inte bara klistra in i en webbläsare var som helst. Det viktiga att tänka på är att en relativ länk kan brytas om man t.ex inte får med alla filer. Har man en index.html och en kontakt.html måste man ha med båda i samma katalog om man länkat dem relativt. Om man istället har en absolut länk så kan man alltid nå den så länge som den som har hand om den absoluta resursen ser till att den finns tillgänglig.

  • Referens mozilla.org

Fråga 8

  • Har man inte HTML5 stöd så kommer okända element sättas till display:inline. För att åtgärda detta kan man sätta display:block på de påverkade elementen i css filen.

  • Referens mozilla.org
